# 📋 FARIIMA Platform Workflow Guide

## روند کامل پلتفرم

### 1️⃣ ثبت‌نام و احراز هویت

#### کارفرما (Client)
```
1. ثبت‌نام با Email/Password یا LinkedIn
2. تکمیل پروفایل:
   - نام کامل
   - شرکت (اختیاری)
   - توضیحات
   - تصویر پروفایل
3. اتصال Wallet (اختیاری - برای پرداخت)
```

#### فریلنسر (Freelancer)
```
1. ثبت‌نام با Email/Password یا LinkedIn
2. تکمیل پروفایل:
   - نام کامل
   - عنوان شغلی
   - بیوگرافی
   - تخصص‌ها (Skills)
   - نرخ ساعتی
   - محل سکونت
   - آپلود رزومه (PDF/DOC)
3. افزودن نمونه کارها (Portfolio):
   - عنوان پروژه
   - توضیحات
   - تصاویر
   - لینک‌های مربوطه
4. اتصال Wallet (برای دریافت پرداخت)
```

```

│ • تخصص‌های مورد نیاز (Skills)   │

پس از ثبت:
✅ پروژه با وضعیت "Open" ذخیره می‌شود
✅ در لیست Find Talent نمایش داده می‌شود
✅ فریلنسرها می‌توانند درخواست دهند
```

### 4️⃣ Apply کردن به پروژه (فریلنسر)

```
فریلنسر → Project Detail → Submit Proposal

فرم Apply شامل:
┌─────────────────────────────────┐
│ 1. Cover Letter (متن معرفی)    │
│    - چرا شما بهترین گزینه‌اید؟ │
│    - تجربیات مرتبط              │
│    - رویکرد شما چیست؟           │
│                                 │
│ 2. قیمت پیشنهادی ($)            │
│    - مبلغ کل یا ساعتی           │
│                                 │
│ 3. زمان تحویل (تعداد روز/هفته)  │
│                                 │
│ 4. فایل‌های پیوست (Portfolio)   │

└─────────────────────────────────┘

وضعیت پس از Apply:
⏳ Pending - منتظر بررسی کارفرما
📩 فریلنسر می‌تواند proposal خود را مشاهده کند
🚫 چت هنوز باز نیست (فقط متن apply ارسال شده)
```

### 6️⃣ قبول درخواست و شروع پروژه

```
کارفرما → Accept Proposal

اتفاقات پس از Accept:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. ✅ وضعیت پروژه → "In Progress"

2. 💬 چت فعال می‌شود:
   - فریلنسر و کارفرما می‌توانند چت کنند
   - امکان ارسال فایل، تصویر، سند
   - نوتیفیکیشن Real-time

3. 💰 Escrow ایجاد می‌شود:
   - کارفرما باید بودجه را واریز کند
   - وجه در Smart Contract Lock می‌شود
   - تا تکمیل پروژه امن نگهداری می‌شود

4. 📊 Milestone ها (اختیاری):
   - تقسیم پروژه به فازها
   - پرداخت مرحله‌ای

5. 🔔 سایر Proposals رد می‌شوند
   - نوتیفیکیشن به سایر فریلنسرها
```

### 9️⃣ تکمیل پروژه

```
فریلنسر → Mark as Complete

1. فریلنسر تحویل می‌دهد:
   ┌─────────────────────────────────┐
   │ • آپلود فایل‌های نهایی         │
   │ • توضیحات تحویل                 │
   │ • درخواست Release Payment       │
   └─────────────────────────────────┘

2. کارفرما بررسی می‌کند:
   ┌─────────────────────────────────┐
   │ ✅ Accept & Release Payment     │
   │    → وجه به Freelancer پرداخت   │
   │    → NFT گواهی صادر می‌شود      │
   │    → پروژه Completed            │
   │                                 │
   │ ❌ Request Revision             │
   │    → بازگشت به In Progress       │
   │    → چت باز می‌ماند             │
   │                                 │
   │ ⚠️  Open Dispute                │
   │    → DAO رای‌گیری می‌کند        │
   └─────────────────────────────────┘
```

### 🔟 پرداخت نهایی

```
Smart Contract → Release Payment

Process:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Escrow → Freelancer: 95%
2. Platform Fee: 5%
3. NFT Certificate → Minted
   • Proof of Work
   • On-chain سابقه
4. Rating & Review
   • هر دو طرف می‌توانند نظر بدهند
```

### 1️⃣1️⃣ Dispute Resolution (در صورت اختلاف)

```
اگر توافق حاصل نشد:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. یکی از طرفین Dispute باز می‌کند:
   ┌─────────────────────────────────┐
   │ • عنوان مشکل                    │
   │ • توضیحات کامل                  │
   │ • آپلود مدارک (Evidence)        │
   │ • درخواست تقسیم بودجه           │
   └─────────────────────────────────┘

2. طرف مقابل پاسخ می‌دهد:
   • ارسال مدارک خود
   • توضیح دیدگاه
   • پیشنهاد تسویه

3. DAO Voting:
   • دارندگان FARI Token رای می‌دهند
   • هر توکن = یک رای
   • مدت رای‌گیری: 7 روز

4. نتیجه:
   • اکثریت تصمیم می‌گیرد
   • تقسیم بودجه طبق رای DAO
   • Escrow Release می‌شود
```

## 📋 Rules & Restrictions

### قبل از Accept:
- ❌ چت غیرفعال
- ✅ فقط متن Apply قابل مشاهده
- ✅ فریلنسر می‌تواند proposal را ویرایش کند
- ✅ کارفرما می‌تواند سوالات عمومی بپرسد (Public Q&A)

### بعد از Accept:
- ✅ چت کامل فعال
- ✅ ارسال فایل و تصویر
- ✅ Real-time نوتیفیکیشن
- ✅ دسترسی به تمام اطلاعات
- ❌ فریلنسر نمی‌تواند proposal را تغییر دهد

### امنیت:
- 🔒 Wallet addresses هیچ‌جا نمایش داده نمی‌شود (جز صفحه Wallets خود کاربر)
- 🔒 اطلاعات تماس Private است
- 🔒 فایل‌ها Encrypted ذخیره می‌شوند
- 🔒 تمام تراکنش‌ها On-chain
